# ТЗ — VP Attack Planner

## 1. Мета продукту

Веб-застосунок для автоматизованого планування масових атак у грі Война Племён (voynaplemyon.com).

Планувальник приймає список сіл своїх гравців з даними по військам, список цілей, налаштування світу і правила планування — і формує синхронізований план відправок для кожного гравця.

**Основний користувач:** лідер/координатор племені, планувальник масів.

**Ключовий результат:** персональний план атак для кожного гравця у форматі BBCode, готовий до відправки в ігровому чаті або месенджері.

## 2. Вхідні дані

### 2.1 CSV-вигрузка своїх гравців

Надається вручну через UI. Один рядок = одне село.

```
Игрок,Координаты,Очки,Копья,Мечи,Топоры,Лазы,ЛК,ТК,Тараны,Каты,Пал,Двор
8Taras8,494|564,9289,13096,3156,0,581,0,0,4,0,0,1
```

Поле `Координаты` має формат `X|Y`. Всі числові поля — кількість юнітів у цьому селі **на момент вигрузки**.

### 2.2 Додаткові дані по гравцях (вводяться вручну)

Збираються координатором окремо, не входять у CSV:

| Поле | Опис |
|---|---|
| `off_paladins` | Кількість **офових паладинів** у гравця (паладин з вміннями атаки). Визначає скільки посилених оффів гравець може відправити. Планер скаже де саме будувати цього паладина. |
| `total_nobles` | Загальна кількість **дворян** які гравець може виставити до початку масу (не прив'язано до конкретного села). Планер сам вирішить в яких селах їх розподілити (max ~6 на село практично). |

### 2.3 Цілі

Вводяться вручну. На кожну ціль задається:
- Координати (`X|Y`)
- ID села (для BBCode-посилань, опціонально)
- Бажаний час прильоту (дата + час + мілісекунди)

### 2.4 Налаштування шаблону масу (динамічні)

Головний конфіг планування. Визначає **рецепт** масу — що і в яких пропорціях відправляти.

#### Оффи

| Параметр | Опис | Приклад |
|---|---|---|
| `paladinOffsPerTarget` | Кількість паладин-оффів на ціль (посилений пробій зі зносом стіни) | 2 |
| `regularOffsPerTarget` | Кількість звичайних оффів на ціль (без паладина) | 3 |
| `splitOff` | Розділяти офф на 2 атаки (тарани в першій, решта в другій) | true/false |

#### Дворяни (noble train)

| Параметр | Опис | Приклад |
|---|---|---|
| `nobleTrainSize` | Скільки дворян в одному паровозі на ціль | 4 |
| `nobleEscortUnit` | Яким юнітом заповнювати до 999 (ЛК, Топори, Мечі, Списники) | `light` |
| `nobleComposition` | Розподіл типів дворянських атак (%) | div нижче |
| `maxNoblesPerVillage` | Практичний максимум дворян в одному селі | 6 |

**Типи дворянських атак і пропорції:**

| Тип | Склад | Іконка башти | % від загалу |
|---|---|---|---|
| `green_strong` | Двір + 999 ЛК (або заданий юніт) | зелений двір | налаштовується |
| `green_weak` | Двір + 50–100 юнітів | зелений двір | налаштовується |
| `orange` | Двір + 1 001–5 000 юнітів | сірий двір | налаштовується |
| `red` | Двір + повний офф (5 001+) | червоний двір | налаштовується |
| `spam_noble` | Двір + мінімум юнітів, ціль — відволікаюча | зелений двір | налаштовується |

#### Спами

| Параметр | Опис |
|---|---|
| `useSpamAttacks` | Чи додавати спам-атаки (без дворян) |
| `spamCountPerTarget` | Кількість спамів на ціль |
| `useSpamNobles` | Чи додавати спам-дворян (двір на відволікаючі цілі) |
| `spamNobleCountPerTarget` | Кількість спам-дворян на ціль |

### 2.5 Налаштування світу

Вводяться один раз, зберігаються. Підтягуються автоматично з API або вручну.

| Параметр | XML-тег | Приклад (ru100) |
|---|---|---|
| `worldSpeed` | `speed` | 2 |
| `unitSpeed` | `unit_speed` | 0.5 |
| `mapSize` | `coord map_size` | 1000 |
| `nightActive` | `night active` | true |
| `nightFrom/To` | `night start_hour / end_hour` | 1 / 8 |
| `snobMaxDist` | `snob max_dist` | 100 |
| `snobIntervalMs` | `commands attack_gap` | 100 |
| `watchtowerEnabled` | `game watchtower` | true |
| `unitTimes` | `get_unit_info` → speed | ram:1800, snob:2100... |

## 3. Основний сценарій

1. Налаштувати світ (код → API або пресет)
2. Завантажити CSV з даними сіл
3. Ввести додаткові дані по гравцях (офові паладини, загальна кількість дворян)
4. Додати цілі з часами прильоту
5. Налаштувати шаблон масу (рецепт атак)
6. Запустити авторозрахунок
7. Перевірити попередження і скоригувати вручну якщо потрібно
8. Отримати персональні плани у BBCode — де що відправляти і де що будувати

## 4. Алгоритм авторозрахунку

### 4.1 Підготовка ресурсів

**Оффи:**
- Визначити `off_villages` — всі села де є тарани (основний юніт оффу)
- Для гравців з `off_paladins > 0` — позначити N найсильніших сіл як `paladin_off`
- Якщо `splitOff = true` — кожне офф-село генерує 2 атаки (тарани + решта)

**Дворяни:**
- Визначити `noble_villages` — всі села в межах `snobMaxDist` від хоча б однієї цілі
- Розподілити `total_nobles` по цих селах (рівномірно або до max `maxNoblesPerVillage`)
- Планер вказує гравцю: "в селі X|Y побудуй N дворян"

**Спами:**
- `spam_villages` — всі інші села (без оффів і дворів), або ті що далеко від цілі

### 4.2 Призначення атак на цілі

Для кожної цілі формується пакет атак відповідно до шаблону масу:

```
1. [paladinOffsPerTarget] паладин-оффів       → найближчі paladin_off села
2. [regularOffsPerTarget] звичайних оффів     → найближчі off_villages
3. [nobleTrainSize] дворів у паровозі         → noble_villages в межах snobMaxDist
   - розподіл по типах: green_strong/green_weak/orange/red — за заданими %
4. [spamCountPerTarget] спамів                → spam_villages (ротація, без повторів)
5. [spamNobleCountPerTarget] спам-дворів      → noble_villages з надлишком дворів
```

Кожна атака = конкретне `fromVillage` + `toTarget` + **точний склад загону**.

### 4.3 Формування складу загону

Для кожної атаки планер визначає точний склад (`composition`) з урахуванням:
- Наявних юнітів у селі (з CSV)
- Типу атаки (green_strong = двір + до 999 ЛК)
- Ліміту ферми (загальна кількість не може перевищити вільну ферму)
- Колір башти = `totalUnits ≤ 1000 → green`, `≤ 5000 → orange`, `> 5000 → red`
- Якщо є двір → іконка дворянина незалежно від решти

### 4.4 Розрахунок часів

```
speedUnit = найповільніший юніт у складі (або визначений типом атаки)
travelSec = distance * unitBaseSec / worldSpeed / unitSpeed
sendTime  = arrivalTime - travelSec
```

Для паровоза дворян:
```
noble[0].arrivalTime = lastOff.arrivalTime + offsetAfterOff_ms
noble[1].arrivalTime = noble[0].arrivalTime + snobIntervalMs
noble[2].arrivalTime = noble[1].arrivalTime + snobIntervalMs
...
```

## 5. Математика

### Відстань

```
dx = min(|x2 - x1|, map_size - |x2 - x1|)
dy = min(|y2 - y1|, map_size - |y2 - y1|)
distance = sqrt(dx² + dy²)
```

### Час в дорозі

```
travel_seconds = distance * unit_base_speed_sec / world_speed / unit_speed
```

де `unit_base_speed_sec` — базова швидкість юніта в секундах на поле (з `get_unit_info`).

### Час відправки

```
send_time = arrival_time - travel_seconds
```

### Послідовність удару по одній цілі

```
1. Основний офф       → головний удар (Топоры + ЛК + Тараны)
2. Дворяни            → приходять ПІСЛЯ останнього оффа (+offset_ms між кожним)
3. Спами              → фейки для перевантаження уваги
// TODO: антисрез тарани (до оффа) та антисрез топори (після дворян) — доп. механіки, поза скоупом MVP
```

### Дворяни після оффа

```
noble_1_arrival = last_off_arrival + offset_ms
noble_2_arrival = noble_1_arrival + noble_gap_ms
noble_3_arrival = noble_2_arrival + noble_gap_ms
...
```

### Тайм-спам

```
randomized_arrival = base_arrival + random(-range_sec, +range_sec) + random(0, 999)ms
```

## 5. Попередження та валідація

| Код | Коли | Дія |
|---|---|---|
| `NIGHT_ARRIVAL` | Час прильоту в 01:00–08:00 | Попередити (захист ×2) |
| `NIGHT_SEND` | Час відправки в нічному вікні | Підсвітити або виключити |
| `SEND_IN_PAST` | Час відправки вже минув | Позначити як неможливу |
| `MORALE_RISK` | Атакуємо набагато меншого гравця | Показати % морального духу |
| `WATCHTOWER_HIT` | Ціль або маршрут в радіусі башти | Підсвітити |
| `NO_VILLAGE` | Немає доступних сіл для ролі | Показати ціль як незакриту |
| `INVALID_COORDS` | Неправильний формат координат | Блокувати розрахунок |
| `DUPLICATE_ASSIGN` | Одне село на двох несумісних ролях | Показати конфлікт |

## 6. Вивід — BBCode план

Основний формат виводу. Групується по гравцю. Кожен гравець отримує свій блок.

```
Гравець «Player1»
Твій план на масс:

[b]АНТИСРЕЗ ТАРАНЫ[/b]
ram [unit]ram[/unit] | 2025-11-23 [b]04:20:40.123[/b] | 09:45:10.000 | 519|555 -> 510|549 | [url=game.php?village=2740&screen=place&target=2722]Attack[/url]

[b]ОФФ[/b]
axe [unit]axe[/unit] | 2025-11-23 [b]05:10:22.456[/b] | 09:45:10.000 | 514|554 -> 510|549 | [url=game.php?village=2704&screen=place&target=2722]Attack[/url]

[b]ЗАХВАТ[/b]
snob [unit]snob[/unit] | 2025-11-23 [b]06:01:04.789[/b] | 09:45:11.000 | 514|554 -> 510|549 | [url=game.php?village=2704&screen=place&target=2722]Attack[/url]

Тобі потрібно відкрити приказ: Player2, Player3
Тобі мають відкрити приказ: Player4
```

Посилання атаки: `game.php?village={from_id}&screen=place&target={target_id}`

## 9. Обмеження

- Застосунок **не відправляє** атаки автоматично
- Всі дані — тільки локально (localStorage), без бекенду
- CSV обробляється тільки на клієнті, нікуди не відправляється
- CORS: `interface.php` може блокуватись — fallback на ручне введення параметрів світу
- Мобільна версія поза скоупом
- Мультикористувацький режим поза скоупом
