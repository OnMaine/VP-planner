# Features — VP Attack Planner

Пріоритети: **P1** = must have (MVP1), **P2** = MVP2–3, **P3** = V1+
Статуси: `[ ]` = не почато, `[~]` = в роботі, `[x]` = готово

---

## P1 — MVP 1 (ядро розрахунку)

### F-01 — Налаштування світу
- [ ] Введення коду світу (напр. `ru100`)
- [ ] Автозавантаження параметрів через `interface.php?func=get_config` (world_speed, unit_speed, map_size, night hours)
- [ ] Автозавантаження швидкостей юнітів через `interface.php?func=get_unit_info`
- [ ] Fallback: ручне введення всіх параметрів якщо CORS блокує
- [ ] Збереження в localStorage

### F-01b — Дані по гравцях (доп. до CSV)
- [ ] Таблиця гравців з полями: `off_paladins` (кількість офових паладинів), `total_nobles` (загальна кількість дворян до масу)
- [ ] Редагується вручну після імпорту CSV
- [ ] Зберігається між сесіями

### F-02 — Імпорт CSV
- [ ] Завантаження файлу через drag & drop або file input
- [ ] Парсинг формату `Игрок,Координаты,Очки,Копья,Мечи,Топоры,Лазы,ЛК,ТК,Тараны,Каты,Пал,Двор`
- [ ] Валідація: формат координат, числові поля, кількість колонок
- [ ] Відображення: кількість сіл, кількість гравців, сумарні юніти
- [ ] Можливість перезавантажити CSV

### F-03 — Одна ціль, один тип атаки
- [ ] Форма: координати цілі (X|Y), час прильоту (datetime до мілісекунд), роль/тип атаки
- [ ] Розрахунок евклідової відстані з wrap-around (тороїдальна карта)
- [ ] Розрахунок travel_seconds = distance × unit_speed_sec / world_speed / unit_speed
- [ ] Розрахунок send_time = arrival_time − travel_seconds
- [ ] Фільтрація: показувати тільки села з ненульовою кількістю потрібного юніта

### F-04 — Таблиця результатів
- [ ] Колонки: Гравець, Координати, Відстань, Час у дорозі, **Час відправки**, Час прильоту, Кількість юнітів
- [ ] Сортування за часом відправки (від найпізнішого до найраннішого)
- [ ] Підсвічування червоним: час відправки вже минув (`SEND_IN_PAST`)
- [ ] Підсвічування жовтим: прильот у нічному вікні (`NIGHT_ARRIVAL`)
- [ ] Чекбокс для виключення конкретного села з плану

## P2 — MVP 2 (ролі, множинні цілі, BBCode)

### F-05 — Ролі сіл
- [ ] Призначення ролі кожному селу: `off`, `half_off`, `noble`, `rampage`, `spam`, `time_spam`
- [ ] Автопризначення ролі на основі складу військ (є дворяни → noble, мало всього → spam, etc.)
- [ ] Збереження призначених ролей між сесіями

### F-05b — Шаблони типів масу
- [ ] При створенні плану обирається тип масу — визначає які хвилі включати
- [ ] Вбудовані типи:
  - **Повний мас** — offs + noble train + спами
  - **Чистий офф** — тільки offs (без дворів і спамів)
  - **Захоплення** — offs + noble train (без спамів)
  - **Спам-мас** — тільки спами (відволікаючий удар)
  - **Кастомний** — ручний вибір хвиль галочками
- [ ] Для кожного типу масу — окремі опції: кількість спамів на ціль, відступ дворів (мс), мінімум юнітів для включення села
- [ ] Збереження кастомних шаблонів з назвою

### F-06 — Множинні цілі

- [ ] Автоматичний підбір сіл на кожну ціль за роллю (ближнє доступне село потрібної ролі)
- [ ] Ручне перепризначення: переназначити село на іншу ціль

### F-07 — Послідовність удару

## Відомі обмеження

- CORS блокує `interface.php` і `map/*.txt.gz` — fallback на ручне введення
- CSV оновлюється вручну, немає real-time даних
- Час відправки рахується локально — годинник браузера має бути точним
- village_id у CSV відсутній — потрібно або підтягати з village.txt, або вводити вручну для BBCode-посилань
