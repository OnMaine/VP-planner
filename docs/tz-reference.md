## 1. Цель продукта

Разработать инструмент для автоматизированного планирования массовых атак в игре «Война племен». Планер должен принимать списки деревень соплеменников, списки целей, настройки мира и правила планирования, после чего формировать синхронизированный план отправок по каждому игроку.

Планер должен:
- рассчитать точное время отправки атак под заданное время прихода
- распределить атакующие деревни по целям с учётом роли деревни и типа атаки
- учесть игровые параметры мира: скорость мира, скорость войск, координаты, типы юнитов
- исключать или подсвечивать ночные отправки
- поддерживать ручную фиксацию захватов: какие дворы должны идти на какие цели
- проверять боевой дух/мораль на основании соотношения очков атакующего и защищающегося
- генерировать BBCode-план, который можно отправить игрокам в игре или в мессенджере
- формировать список игроков, которым нужно открыть приказы, и список игроков, которые должны открыть приказы пользователю

## 2. Контекст и исходные данные

В племени есть набор собственных деревень, заранее разделённых по назначению. На противоположной стороне есть список вражеских целей, также разделённый по типу атаки или назначению.

| Категория входных данных | Описание |
|---|---|
| Оффы | Деревни, из которых идут основные атакующие удары |
| 1/2 оффа | Деревни с половинными оффами, могут использоваться для добивания, раскатки, поддержки основного удара |
| Дворы | Деревни с дворянами/снобами для захвата |
| Отряды раскатки | Деревни с катапультами/таранами/оффом для разрушения целей |
| Спамы | Деревни, из которых идут массовые ложные атаки |
| Тайм-спамы | Спамы с заданным окном времени и/или рандомизацией |
| Цели | Вражеские деревни по каждому типу атаки: офф, захват, раскатка, спам, тайм-спам |

## 3. Роли и сценарии использования

| Роль | Что делает в системе |
|---|---|
| Планировщик | Загружает деревни, цели, настройки мира, задаёт правила и запускает расчёт |
| Игрок-участник | Получает персональный план отправок и список необходимых действий |
| Администратор племени | Поддерживает справочники скоростей, форматы импорта, шаблоны планов |

### 3.1 Основной пользовательский сценарий

1. Планировщик выбирает мир или вручную вводит скорость мира и скорость войск
2. Загружает список собственных деревень соплеменников с типами: офф, полуофф, двор, раскатка, спам, тайм-спам
3. Загружает список целей по каждой категории
4. Задаёт время прихода массы или несколько волн прихода
5. Указывает правила: исключать ночные отправки, сколько спамов на цель, как ставить дворы после оффа, учитывать ли сторожевые башни
6. Запускает расчёт
7. Проверяет предупреждения: ночные отправки, мораль, пересечение сторожевых башен, нехватка деревень/войск
8. Экспортирует персональные планы для игроков

## 7. Математика и алгоритмы

### 7.1 Разбор координат

```python
def parse_coords(coords: str) -> tuple[int, int]:
    # Поддерживать форматы 519|555 и 519/555
    coords = coords.strip().replace('/', '|')
    x_str, y_str = coords.split('|')
    return int(x_str), int(y_str)
```

### 7.2 Расстояние

```python
distance = sqrt((x1 - x2) ** 2 + (y1 - y2) ** 2)
```

> Некоторые внутренние скрипты могут использовать упрощённую модель `max(abs(dx), abs(dy))`, но для боевого планера правильнее держать формулу расстояния параметризуемой. По умолчанию рекомендуется евклидово расстояние.

### 7.3 Время в пути

```python
travel_time_seconds = distance * unit_time_seconds_per_tile / world_speed / unit_speed
```

### 7.4 Время отправки

```python
send_time = arrival_time - timedelta(seconds=travel_time_seconds)
```

### 7.5 Ночные отправки

```python
def is_in_night_block(dt, night_from, night_to):
    t = dt.time()
    if night_from <= night_to:
        return night_from <= t <= night_to
    return t >= night_from or t <= night_to
```

Поведение при попадании в ночной блок должно настраиваться:
- `mark_only` — только подсветить проблемную отправку
- `exclude` — исключить такую деревню из кандидатов
- `shift_arrival` — сдвинуть приход всей волны или конкретной атаки
- `manual_review` — вывести предупреждение и оставить решение за планировщиком

```python
def pick_best_village(candidates, target, used_village_ids, world, unit):
    available = [v for v in candidates if v.id not in used_village_ids]
    available.sort(key=lambda v: calc_distance(v.coords, target.coords))
```

## 9. Боевой дух / мораль

```python
def morale_status(attacker_points: int, defender_points: int):
    if defender_points <= 0:
        return "UNKNOWN"
    ratio = attacker_points / defender_points
    if ratio >= 1.5:
        return "OK"
    if ratio >= 1.0:
        return "MEDIUM"
    return "HIGH"
```

## 10. Сторожевые башни

| Режим | Описание |
|---|---|
| Маркировка цели | Если цель находится в радиусе башни, пометить как `watchtower_hit` |
| Маркировка отправителя | Если отправитель/цель находятся в зоне башни, подсветить риск |
| Исключение | При включённой настройке `avoid_watchtower` исключать такие маршруты из автоподбора |
| Ручной режим | Оставлять предупреждение без автоматического исключения |

```python
def watchtower_hit(target_coords, tower_coords, radius):
    d = calc_distance(target_coords, tower_coords)
    return d <= radius
```

> Для MVP достаточно проверки расстояния от цели до башни. Проверку пересечения маршрута с радиусом башни можно вынести в post-MVP.

## 11. Формирование выходного плана

### 11.1 Группировка по игрокам

```python
plans_by_player = {}

for attack in attacks:
    player = attack.from_village.player
    plans_by_player.setdefault(player, []).append(attack)

for player in plans_by_player:
    plans_by_player[player].sort(key=lambda a: a.send_time)
```

### 11.2 Формат блока игрока

```
Игрок «{player}»
Твой план на масс:
{bbcode_plan}

Тебе нужно открыть приказы игрокам: {open_orders_to}
Тебе должны открыть приказы игроки: {orders_to_receive_from}
```

### 11.3 Логика открытия приказов

```python
def build_open_orders(attacks):
    open_to = {}
    receive_from = {}
    for attack in attacks:
        sender = attack.from_village.player
        target_owner = attack.target.player
        if sender == target_owner:
            continue
        open_to.setdefault(sender, set()).add(target_owner)
        receive_from.setdefault(target_owner, set()).add(sender)
    return open_to, receive_from
```

```python
def format_attack_bbcode(attack):
    send_date = attack.send_time.strftime("%Y-%m-%d")
    send_time = attack.send_time.strftime("%H:%M:%S.%f")[:12]
    arr = attack.arrival_time.strftime("%Y-%m-%d %H:%M:%S.%f")[:23]

        f"{arr} | "
        f"{attack.from_coords} -> {attack.to_coords} | "
```

## 14. MVP-код на Python

```python
from dataclasses import dataclass
from datetime import datetime, timedelta, time
from math import sqrt
from typing import Optional

@dataclass
class Village:
    village_id: int
    coords: str
    player: str
    role: str

@dataclass
class Target:
    target_id: int
    coords: str
    player: str
    target_type: str
    arrival_time: datetime

@dataclass
class WorldSettings:
    world_speed: float
    unit_speed: float
    unit_times: dict
    night_enabled: bool = False
    night_from: Optional[time] = None
    night_to: Optional[time] = None

@dataclass
class Attack:
    village: Village
    target: Target
    unit: str
    send_time: datetime
    arrival_time: datetime
    distance: float
    travel_seconds: float
    warnings: list

def parse_coords(coords: str):
    coords = coords.strip().replace('/', '|')
    x, y = coords.split('|')
    return int(x), int(y)

def calc_distance(a: str, b: str) -> float:
    ax, ay = parse_coords(a)
    bx, by = parse_coords(b)
    return sqrt((ax - bx) ** 2 + (ay - by) ** 2)

def is_in_night_block(dt: datetime, start: time, end: time) -> bool:
    current = dt.time()
    if start <= end:
        return start <= current <= end
    return current >= start or current <= end

def build_attack(village: Village, target: Target, unit: str, world: WorldSettings) -> Attack:
    distance = calc_distance(village.coords, target.coords)
    unit_time = world.unit_times[unit]
    travel_seconds = distance * unit_time / world.world_speed / world.unit_speed
    send_time = target.arrival_time - timedelta(seconds=travel_seconds)
    warnings = []
    if world.night_enabled and is_in_night_block(send_time, world.night_from, world.night_to):
        warnings.append('NIGHT_SEND')
    if send_time < datetime.now():
        warnings.append('SEND_TIME_IN_PAST')
    return Attack(
        village=village,
        target=target,
        unit=unit,
        send_time=send_time,
        arrival_time=target.arrival_time,
        distance=distance,
        travel_seconds=travel_seconds,
        warnings=warnings,
    )

def format_attack_bbcode(attack: Attack) -> str:
    send_date = attack.send_time.strftime('%Y-%m-%d')
    send_time = attack.send_time.strftime('%H:%M:%S.%f')[:12]
    arrival = attack.arrival_time.strftime('%Y-%m-%d %H:%M:%S.%f')[:23]
    link = f"game.php?village={attack.village.village_id}&screen=place&target={attack.target.target_id}"
    return (
        f"{attack.unit} [unit]{attack.unit}[/unit] | "
        f"{send_date} [b]{send_time}[/b] | "
        f"{arrival} | "
        f"{attack.village.coords} -> {attack.target.coords} | "
        f"[url={link}]Attack[/url]"
    )

if __name__ == '__main__':
    world = WorldSettings(
        world_speed=1.0,
        unit_speed=1.0,
        unit_times={'ram': 1800, 'axe': 1080, 'snob': 2100},
        night_enabled=True,
        night_from=time(0, 0),
        night_to=time(7, 0),
    )
    village = Village(2740, '519|555', 'Player1', 'off')
    target = Target(2722, '510|549', 'Enemy1', 'off', datetime(2025, 11, 23, 9, 45, 10))
    attack = build_attack(village, target, 'ram', world)
    print(format_attack_bbcode(attack))
    print(attack.warnings)
```

## 15. Валидация и ошибки

| Ошибка/предупреждение | Когда возникает | Что делать |
|---|---|---|
| `INVALID_COORDS` | Координаты не соответствуют формату `xxx\|yyy` или `xxx/yyy` | Не запускать расчёт, показать строку с ошибкой |
| `UNKNOWN_UNIT` | Юнит отсутствует в справочнике скоростей | Попросить выбрать юнит или обновить пресет мира |
| `NO_AVAILABLE_VILLAGE` | Нет доступных деревень нужной роли | Показать цель как не закрытую |
| `NIGHT_SEND` | Отправка попадает в ночной интервал | Подсветить или исключить в зависимости от режима |
| `SEND_TIME_IN_PAST` | Время отправки уже прошло | Пометить как невозможную отправку |
| `DUPLICATE_VILLAGE_ASSIGNMENT` | Одна деревня назначена на несколько несовместимых атак | Показать конфликт |
| `WATCHTOWER_HIT` | Цель или маршрут попадает в радиус башни | Подсветить или исключить |
| `MORALE_HIGH_RISK` | Боевой дух ниже порога | Показать предупреждение планировщику |

## 20. Критерии приёмки

- Планер корректно рассчитывает время отправки для известных тестовых пар деревня-цель
- В итоговом плане каждая атака имеет отправителя, цель, юнит, время отправки, время прихода и ссылку
- Для каждого игрока формируется отдельный блок плана
- Ночные отправки подсвечиваются или исключаются согласно выбранной настройке
- Ручные назначения дворов не перетираются автоподбором
- Дворы приходят после последнего оффа с заданным отступом
- BBCode копируется и сохраняет структуру блоков
- CSV/JSON экспорт содержит те же атаки, что и BBCode
- Ошибки входных данных не ломают расчёт, а показываются в понятном списке предупреждений
